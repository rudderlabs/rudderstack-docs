# How to Filter Events using Different Methods

RudderStack provides different methods to filter out events before sending them to the specified destinations. The below table explains these methods in detail:

| Method   | Description        | Supported mode | Applicable |
| :--------- | :----------- | :----------- |:----------- |
| [User transformations](https://www.rudderstack.com/docs/transformations/) | Create transformations to filter out certain event names, types, or any distinguishing factor among them. | Cloud mode |  [Event Stream](https://www.rudderstack.com/docs/stream-sources/), [Reverse ETL](https://www.rudderstack.com/docs/reverse-etl/) and [Cloud Extract](https://www.rudderstack.com/docs/cloud-extract-sources/) |
| [Event filtering tool](https://www.rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/event-filtering/) | Specify which events should be discarded or allowed to flow through, by whitelisting or blacklisting them from the UI. | Device mode | [All SDKs](https://www.rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/) |
| [Send events to selective destinations](https://www.rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/rudderstack-javascript-sdk/#loading-selective-destinations) | Send your event data only to the selective destinations. |Device mode | [JavaScript SDK](https://www.rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/rudderstack-javascript-sdk/) |
| [Filter out selective destinations](https://www.rudderstack.com/docs/user-guides/how-to-guides/how-to-filter-selective-destinations/)   | Filter out specific destinations while sending your event data. | Cloud and Device mode | [JavaScript SDK](https://www.rudderstack.com/docs/stream-sources/rudderstack-sdk-integration-guides/rudderstack-javascript-sdk/) |
