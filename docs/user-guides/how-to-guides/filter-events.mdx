---
title: "How to Filter Events using Different Methods"
description: An easy-to-follow guide on filtering the events before sending them to your specified destinations in RudderStack.
---

RudderStack provides different filtering methods before sending the events to your specified destinations. The following table explains these methods in detail:

| Method     | Description    | Connection mode | Applicable |
| :--------- | :----------- | :----------- |:----------- |
| <Link to="/features/transformations/">User Transformations</Link> | Create transformations to filter out certain event names, types, or any distinguishing factors among them. | Cloud mode | <ul><li><Link to="/sources/cloud-apps/">Cloud Apps</Link></li><li><Link to="/sources/reverse-etl/">Reverse ETL</Link></li><li><Link to="/sources/extract/">Cloud Extract</Link></li></ul> |
| <Link to="/sources/sdks/event-filtering/">Client-side Event Filtering</Link> | Specify which events should be discarded or allowed to flow through by allowlisting or denylisting them in the RudderStack dashboard, while setting up the destination. | Device mode | <Link to="/sources/sdks/">All SDKs</Link> |
| <Link to="/sources/sdks/rudderstack-javascript-sdk//#loading-selective-destinations">Sending events to selective destinations</Link> | Send your events only to the selective destinations by stopping the SDK from loading other device mode integrations. |Device mode | <Link to="/sources/sdks/rudderstack-javascript-sdk/">JavaScript SDK</Link> |
| <Link to="/user-guides/how-to-guides/how-to-filter-selective-destinations/">Filtering selective destinations</Link>   | Filter out selective destinations by stopping RudderStack from sending certain events to them. | Cloud mode and device mode | <ul><li><Link to="/sources/sdks/rudderstack-javascript-sdk/#filtering-selective-destinations">JavaScript SDK</Link></li><li><Link to="/sources/sdks/rudderstack-android-sdk/#enablingdisabling-events-for-specific-destinations">Android SDK</Link></li><li><Link to="/sources/sdks/rudderstack-ios-sdk/#enablingdisabling-events-for-specific-destinations">iOS SDK</Link></li></ul> |
