This document covers the necessary steps to migrate from [Snowplow](https://snowplow.io/) to RudderStack and replace your instrumentation code so that you can start using the RudderStack SDKs to track your events with minimal code changes.

## Setting up RudderStack dashboard

1. [Sign up](https://app.rudderstack.com/signup) for the RudderStack dashboard.
2. Set up the source-destination connections in the dashboard to facilitate the event data flow. For more information on setting up the connections, refer to the <Link to="/get-started/quickstart/">Quickstart</Link> guide.

<div class="successBlock">
Note the <Link to="/resources/glossary/#write-key">source write key</Link> and the <Link to="/dashboard-guides/overview/#data-plane-url">data plane URL</Link> as they will be used in the next section.
</div>

<h3>SubjectConfiguration</h3>

The <a href="https://docs.snowplow.io/docs/collecting-data/collecting-from-own-applications/mobile-trackers/tracking-events/"><code class="inline-code">SubjectConfiguration</code></a> call can be used to capture basic information about the user and the app which is attached on all the events as context entity.

Snowplow does not support the <code class="inline-code">identify</code> call. Hence, RudderStack calls the <code class="inline-code">identify</code> method if you provide the <code class="inline-code">SubjectConfiguration</code> while initializing the SDK. If not provided, a user is not identified. 

The <code class="inline-code">userId</code> field is mapped to RudderStack's <code class="inline-code">userId</code>. All the other properties are mapped to RudderStack's <code class="inline-code">traits</code> object.

<div class="warningBlock">
Note that <code class="inline-code">userId</code> is a mandatory field. If not provided, the <code class="inline-code">identify</code> call is ignored.
</div>

## Sending event data

RudderStack maps the event data from Snowplow classes to the `track` or `page`/`screen` calls.

### Identifying users

<Tabs>
  <TabList>
    <Tab>iOS/Android</Tab>
    <Tab>JavaScript</Tab>
  </TabList>
    <TabPanels>
      <TabPanel>
For the Mobile SDKs, Snowplow's <a href="https://docs.snowplow.io/docs/collecting-data/collecting-from-own-applications/mobile-trackers/tracking-events/"><code class="inline-code">SubjectConfiguration</code></a> object can be used to identify the users. Refer to the <Link to="/user-guides/migration-guides/snowplow-migration-guide/#defining-the-configuration-objects">Defining the configuration objects</Link> section for more information.
      </TabPanel>
      <TabPanel>

Snowplow supports identifying a user by passing the <code class="inline-code">setUserId</code> property.

A sample call including the <code class="inline-code">setUserId</code> property will trigger the <Link to="/event-spec/standard-events/identify"><code class="inline-code">identify</code></Link> call in RudderStack JavaScript SDK:

<span>

```javascript
rs.snowplowAdapter('setUserId', 'john.doe@email.com', {
  firstName: 'John',
  lastName: 'Doe',
  city: 'Barcelona',
});
```
</span>
In the above example, <code class="inline-code">john.doe@email.com</code> is the <code class="inline-code">userId</code> and the <code class="inline-code">firstName</code>, <code class="inline-code">lastName</code>, and <code class="inline-code">city</code> are the user traits.
      </TabPanel>
    </TabPanels>
</Tabs>

```javascript
rs.snowplowAdapter('trackSelfDescribingEvent', {
  event: {
    data: {
      action: 'order completed',
      category: 'FCW',
      product_id: 'P1100DFG9766',
      revenue: 30,
      currency: 'USD',
      user_actual_id: 12345,
    },
  },
}); 
```
</span>
