[Monday](https://monday.com/) is a popular workflow management tool. It lets you create efficient cross-team workflows and automations for better collaboration and maximum productivity.

RudderStack supports Monday as a destination to which you can seamlessly send your event data.

<div class="infoBlock">
Find the open source transformer code for this destination in the <a href="https://github.com/rudderlabs/rudder-transformer/tree/master/v0/destinations/monday">GitHub repository</a>.
</div>

## Getting started

Before configuring Monday as a destination in RudderStack, verify if the source platform is supported by Monday by referring to the table below:

| **Connection Mode** | **Web** | **Mobile** | **Server** |
| :--- | :--- | :--- | :--- |
| **Device mode** | - | - | - |
| **Cloud mode** | **Supported**  | **Supported** | **Supported** |

<div class="infoBlock">
To know more about the difference between cloud mode and device mode in RudderStack, refer to the <Link to="/destinations/rudderstack-connection-modes/">RudderStack Connection Modes</Link> guide.
</div>

Once you have confirmed that the source platform supports sending events to Monday, follow these steps:

1. From your [RudderStack dashboard](https://app.rudderstack.com/), add a source. Then, from the list of destinations, select **Monday**.
2. Assign a name to the destination and click **Continue**.

### Connection settings

To successfully configure Monday as a destination, you will need to configure the following settings:

<img src="../../assets/event-stream-destinations/monday-connection-settings.png" alt="Monday connection settings"/>

- **API Token**: Enter the API token from Monday dashboard.
- **Board ID**: Enter the board ID from Monday dashboard.
- **Group Title**: Enter the group title of your board.
- **Columns to property mapping**: Enter the column names from your board and the properties you want them to be mapped to.

<div class="infoBlock">
For more information on getting these details, refer to the <a href="#faq">FAQ</a> section below.
</div>

### Client-side events filtering

This setting lets you specify the event names which should be allowed to flow through to Monday. Any other event is discarded.

<img src="../../assets/event-stream-destinations/monday-connection-settings-2.png" alt="Monday connection settings"/>

## Track

In Monday, you can use the <Link to="/event-spec/standard-events/track">`track`</Link> call to create an item in your board. You also need to provide the **Board ID** and **column to property mapping** in the RudderStack dashboard to create an item. 

<div class="infoBlock">
The column names provided in the mapping should be existing in your board. Only the column names provided in the mapping will be updated.
</div>


The following column types can be updated while creating a new item on the board:

- Status 
- Numbers 
- Text 
- Checkbox
- Country 
- Email 
- Item name 
- Link 
- Location 
- Long Text 
- Phone 
- Rating 
- World Clock

A sample `track` call is shown below:

```javascript
rudderanalytics.track("event", {
  status: "done",
  value: 10000,
  close_probability: "90%",
  location: "New York, USA"
});
```

### Property mapping

RudderStack maps the following event properties to the corresponding Monday properties:

| RudderStack property | Monday property | Presence |
| :-----| :------| :-------|
| `properties.name` | `item_name` | Required |
| `properties.countryCode` | `countryCode` | Required, if `country` column needs to be updated |
| `properties.emailText` | `text`  | Optional |
| `properties.latitude` | `latitude`  | Required, if `location` column needs to be updated |
| `properties.longitude` | `longitude`  | Required, if `location` column needs to be updated |
| `properties.countryShortName` | `countryShortName`  | Required, if `phone` column needs to be updated |
| `properties.linkText` | `text`  | Optional |

## FAQ

### Where can I find the Monday API token?

To get the Monday API token, follow these steps:

1. Log into the [Monday dashboard](https://auth.monday.com/auth/login_monday).
2. Click on your profile in the left navigation bar.
3. Select **Developer** > **My Access Tokens**, as shown:

<img src="../../assets/event-stream-destinations/monday-api-token.png" alt="Monday API token in the dashboard"/>

### Where can I find the Board ID, Group Title and column names in Monday dashboard?

Follow these steps to find the above mentioned attributes:

**Board ID**

1. Log into the [Monday dashboard](https://auth.monday.com/auth/login_monday).
2. Go to **Work management** in the left navigation bar and select your board. The board ID is displayed in the address bar.

**Group title**

Group title is the name of your the groups in the board, for example, **Active deals** and **Closed Won** are the group titles in the below image.

**Column names**

The column titles in your board are the column names, for example, **Stage**, **Owner**, **Contacts**, and **Accounts** are the column names in the below image.

<img src="../../assets/event-stream-destinations/monday-dashboard.png" alt="Monday dashboard"/>
