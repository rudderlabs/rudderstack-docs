## Identify

You can use the <Link to="/event-spec/standard-events/identify/">`identify`</Link> call to add a contact in Sendinblue. If a contact already exists, RudderStack updates the contact details.


A sample `identify` call is shown below:

```javascript
rudderanalytics.identify(
  "rapheal_morrow@example.com", {
    firstName: "Raphael",
    lastName: "Morrow",
    phone: "+919407339881",
    payment: 2,
    age: 21,
    location: "Mumbai",
  }, {
    externalId: [{
        type: "includeListIds",
        id: 1,
      },
      {
        type: "includeListIds",
        id: 2,
      },
    ],
  }
);
```

| :-----| :--------| :------|

| `traits.phone` <br/> `context.traits.phone` <br/> `properties.phone` <span style="color: #4D4DFF;font-size:12px;">Required, if email is not passed.</span>| `attributes.SMS`  |  String |
| `context.traits.firstName` <br/> `context.traits.firstname` <br/> `context.traits.first_name` <br/> `traits.firstName` <br/> `traits.firstname` <br/> `traits.first_name`  | `attributes.FIRSTNAME` | String | 
| `context.traits.lastName` <br/> `context.traits.lastname` <br/> `context.traits.last_name` <br/> `traits.lastName` <br/> `traits.lastname` <br/> `traits.last_name`  | `attributes.LASTNAME` | String | 
| `event` <br/> <span style="color: #4D4DFF;font-size:12px;">Required</span> | `event` | String |
| `context.traits` <br/> `traits` (excluding `firstname`, `lastname`, and `phone`) | `attributes` | Object |
| `integrations.sendinblue.emailBlacklisted`| `emailBlacklisted` | boolean |
| `integrations.sendinblue.smsBlacklisted`| `smsBlacklisted` | boolean |
| `context.externalId.id` (when `context.externalId.type` = `includeListIds`) | `listIds` | int64 Array |

## Track

You can use the <Link to="/event-spec/standard-events/track">`track`</Link> call to:

- **[Track event](https://tracker-doc.sendinblue.com/reference/trackevent-3)**: Lets you capture events on your website, along with optional extra information about the event.
- **[Track Link](https://tracker-doc.sendinblue.com/reference/tracklink-3)**: Lets you capture clicks on your website, along with optional extra information about the click.

A sample `track` call is as shown below:

```javascript
// track events
rudderanalytics.track("Card Created", {
  id: "a4123c72-c6f7-4d8e-b8cd-4abb8a807891",
  products: [{
      product_id: 1234,
      product_name: "Track Pants",
      amount: 1,
      price: 220,
    },
    {
      product_id: 5768,
      product_name: "T-Shirt",
      amount: 5,
      price: 1058,
    },
  ],
});

//track link
rudderanalytics.track("trackLink", {
  link: "https://mail.google.com/mail/u/0/#inbox",
  subject: "Email confirmation",
});
```

## Page

You can use the <Link to="/event-spec/standard-events/page/">`page`</Link> call to send any page-related information to Sendinblue.

A sample `page` call is shown below:

```javascript
rudderanalytics.page("Home")
```

### Supported mappings

The following table details the property mappings between RudderStack and Sendinblue:

| RudderStack property | Sendinblue property | Data type |
| :-----| :--------| :------|
| `context.traits.email` <br/> `properties.email`<br/> `context.traits.phone` <br/> `properties.phone` <span style="color: #4D4DFF;font-size:12px;">Required</span> | `email` | String |
| `properties.title` <br/> `context.page.title`| `properties.ma_title`  | String |
| `properties.url` <br/> `context.page.url` | `page` | String |  
| `properties.path` <br/> `context.page.path`| `properties.ma_path`  |  String |
| `properties.referrer` <br/> `context.page.referrer`| `properties.ma_referrer` | String |
| `properties.name` <br/> **Page Viewed** | `properties.sib_name` | String |
| `properties` (excluding `title`, `url`, `path`, `referrer`, and `name`) | `properties` | Object |
