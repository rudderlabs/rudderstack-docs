The `screen` call lets you record whenever your user views their mobile screen with any additional relevant information about the screen.

<div class="successBlock">
The <code class="inline-code">screen</code> call is the mobile equivalent of the <code class="inline-code">page</code> call.
</div>

## Sample Payload

A sample payload of a `screen` call with most of the <Link to="/event-spec/standard-events/common-fields/">Common fields</Link> removed is as shown:

```javascript
{
  "type": "screen",
  "name": "Main",
  "properties": {
    "prop_key": "prop_value"
  }
}
```

The corresponding event that generates the above payload via the <Link to="/sources/event-streams/sdks/rudderstack-ios-sdk/">iOS SDK</Link> is as shown:

```javascript
[[RSClient sharedInstance] screen:@"Main"
                properties:@{@"prop_key" : @"prop_value"}];
```

## Screen Fields

Apart from the <Link to="/event-spec/standard-events/common-fields/">Common fields</Link>, the `screen` call accepts the following fields:

| **Field**    | **Type** | **Presence** | **Description**                                                                                                                              |
| :----------- | :------- | :----------- | :------------------------------------------------------------------------------------------------------------------------------------------- |
| `name`       | String   | Optional     | The name of the screen.                                                                                                                      |
| `properties` | Object   | Optional     | Includes the properties of the screen such as the `url`, `referrer`, etc. For more more information, check the <Link to="#properties">Properties</Link> section below. |

## Properties

Properties are additional information that describe the viewed screen.

RudderStack has reserved some standard properties listed in the table below and handles them in special ways.

| **Property** | **Type** | **Description**                                                                    |
| :----------- | :------- | :--------------------------------------------------------------------------------- |
| `name`       | String   | You can tag each screen with a `name`. This is a reserved property for future use. |
